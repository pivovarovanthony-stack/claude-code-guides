Claude Code Настройка

Пошаговая инструкция: как настроить Claude Code для любого проекта. Простым языком, без лишнего.

1

Подготовка

Перед началом тебе нужно:

Что должно быть установлено
  1. VS Code — скачай с code.visualstudio.com если ещё нет
  2. Расширение Claude Code — открой VS Code, нажми на иконку расширений (квадратики слева), напиши "Claude Code" в поиске, нажми "Install"
  3. Аккаунт Claude — нужен Claude Pro или Max (платная подписка на claude.ai)
i Claude Code — это ИИ-помощник для программирования. Он читает твой проект, понимает код, и может писать/редактировать файлы, запускать команды, и помогать с любыми задачами.
2

Первый запуск

  1. Открой VS Code и открой папку своего проекта (File → Open Folder)
  2. Открой панель Claude Code — она появится в боковой панели после установки расширения. Кликни на иконку Claude слева.
  3. Войди в аккаунт — Claude попросит авторизоваться. Следуй инструкциям на экране.
  4. Готово! Теперь ты можешь писать Claude что делать прямо в чате.
* Совет: Напиши Claude /init — он автоматически просканирует твой проект и создаст файл CLAUDE.md с базовыми настройками.
3

Файл CLAUDE.md — Мозг проекта

Это самый важный файл. Claude читает его в начале каждой сессии. Здесь ты описываешь свой проект и правила работы.

Главные правила
  • Пиши коротко — до 200 строк. Длинные файлы Claude хуже воспринимает
  • Пиши конкретно: "Используй 4 пробела для отступов" вместо "Форматируй код красиво"
  • Указывай команды: "Запусти npm test перед коммитом"

Какие файлы бывают

ФайлДля чегоКто видит
CLAUDE.md Инструкции для проекта Все кто работают над проектом
CLAUDE.local.md Твои личные предпочтения Только ты

Шаблон — скопируй и измени под свой проект

# Название проекта

- Backend: Node / Python / PHP (что используешь)

Вот как выглядит правильно настроенный проект:

мой-проект/ ├── CLAUDE.md # Инструкции для Claude ├── CLAUDE.local.md # Твои личные настройки │ ├── .claude/ # Папка конфигурации Claude │ ├── settings.json # Общие настройки проекта │ │ │ ├── rules/ # Правила кодирования │ │ ├── common/ # Общие правила (для всех языков) │ │ │ ├── coding-style.md │ │ │ ├── testing.md │ │ │ └── security.md │ │ └── [язык]/ # Правила для конкретного языка │ │ │ ├── skills/ # Свои команды-навыки │ └── agents/ # Специальные помощники │ └── src/ # Твой код
i Не нужно создавать всё сразу. Начни с CLAUDE.md — остальное можно добавлять по мере необходимости.
5

Правила кодирования

Правила — это файлы в папке .claude/rules/. Claude загружает их автоматически и следует им при написании кода.

Как это работает

  • Правила в common/ действуют для всех файлов
  • Правила в папке языка (например python/) действуют только для файлов этого языка
  • Если правила конфликтуют — правило языка важнее

Готовые наборы правил

Можно скачать проверенные правила для своего языка. Открой терминал (Terminal → New Terminal в VS Code) и выполни:

# 1. Скачать набор правил (временно)
git clone --depth 1 https://github.com/affaan-m/everything-claude-code.git /tmp/ecc

# 2. Скопировать нужные правила в свой проект
cp -r /tmp/ecc/rules/common .claude/rules/

# 3. Добавить правила для своего языка (выбери нужные):
cp -r /tmp/ecc/rules/python .claude/rules/       # для Python
cp -r /tmp/ecc/rules/typescript .claude/rules/    # для TypeScript/JavaScript
cp -r /tmp/ecc/rules/php .claude/rules/           # для PHP

# 4. Удалить временную папку
rm -rf /tmp/ecc

Навыки (Skills) — Свои команды

Навыки — это твои собственные команды, которые ты вызываешь через /имя-навыка. Например: /deploy, /seo-audit, /create-post.

Как создать навык

  1. Создай папку: .claude/skills/название-навыка/
  2. Создай файл SKILL.md внутри этой папки
  3. Опиши что навык делает (Claude будет следовать этим инструкциям)

Пример навыка

# Файл: .claude/skills/create-post/SKILL.md

---
name: create-post
description: Создать новый пост для блога
---

Создай новый пост для блога:

1. Спроси тему поста
2. Напиши заголовок и мета-описание
3. Напиши текст поста в формате Markdown
4. Сохрани в папку content/blog-posts/

Теперь можно написать Claude /create-post и он выполнит эти шаги.

Настройки и разрешения

В файле .claude/settings.json можно указать что Claude может и не может делать.

Пример настроек

{
  "autoMemoryEnabled": true,
  "permissions": {
    "deny": [
      "Bash(rm -rf *)",
      "Edit(.env)"
    ]
  }
}

Что значат эти настройки

НастройкаЧто делает
autoMemoryEnabled: trueClaude запоминает важные вещи между сессиями
deny: ["Bash(rm -rf *)"]Запрещает Claude удалять все файлы (защита)
deny: ["Edit(.env)"]Запрещает редактировать файл с паролями
! Важно: Настройки deny — это защита. Claude не сможет случайно удалить твои файлы или изменить пароли.

Память Claude

Claude автоматически запоминает важные вещи о проекте. Это работает между сессиями — закрыл VS Code, открыл заново, а Claude помнит.

Что запоминает

  • Какие команды нужны для сборки проекта
  • Какие баги ты исправлял и как
  • Твои предпочтения по стилю кода
  • Структуру проекта

Можно попросить запомнить явно

"Запомни что наш сервер работает на порту 3000"
"Запомни что для этого проекта мы используем Tailwind CSS"
"Запомни что все картинки кладём в папку assets/images/"
* Память хранится на твоём компьютере, не в облаке. Она привязана к папке проекта.

MCP — Подключение внешних сервисов

MCP (Model Context Protocol) позволяет Claude работать с внешними сервисами: Google Sheets, Notion, Figma и другими.

Когда это нужно

MCP нужен когда ты хочешь чтобы Claude мог:

  • Читать и писать в Google Sheets
  • Работать с задачами в Notion
  • Смотреть дизайны в Figma
  • Подключаться к базам данных
i Это продвинутая функция. Если ты только начинаешь — пропусти этот раздел. Вернёшься когда понадобится.

Как подключить

В терминале VS Code (Terminal → New Terminal):

# Пример: подключить Google Sheets
claude mcp add -s user google-sheets -- npx -y @anthropic-ai/mcp-server-google-sheets
! Не подключай больше 10 сервисов одновременно — каждый занимает память Claude и замедляет работу.

Чеклист настройки

Отмечай галочками по мере выполнения (сохраняется в браузере):

Обязательно (сделай первым)
  • Установить VS Code
  • Установить расширение Claude Code в VS Code
  • Войти в аккаунт Claude (нужна подписка Pro или Max)
  • Открыть папку проекта в VS Code
  • Создать файл CLAUDE.md с описанием проекта
  • Проверить что Claude видит твой CLAUDE.md (написать ему "Какой это проект?")
Рекомендуется (сделай потом)
  • Создать CLAUDE.local.md с личными настройками
  • Создать .claude/settings.json с защитными правилами
  • Скачать правила кодирования для своего языка
  • Проверить что автопамять включена
Продвинутое (когда будешь готова)
  • Создать свои навыки (skills)
  • Подключить MCP-сервисы (Google Sheets, Notion и т.д.)
  • Настроить специальных агентов для код-ревью
